# 清理简化代码

使用 `git diff --cached --name-only` 和 `git diff --name-only` 获取本次修改的文件，逐文件审查以下问题：

## 审查清单

### 冗余代码
- [ ] 重复逻辑（相同代码出现多次，应封装复用）
- [ ] 可用现有工具函数替代的手写逻辑

### 过度设计
- [ ] 只使用一次的抽象（函数、组件、composable）
- [ ] 处理不会发生的边界情况
- [ ] 为假设的未来需求预留的代码

## 输出格式

```
📍 path/to/file.ts:行号
🏷️ [冗余代码|过度设计]
📝 问题描述
💡 修复建议
```
